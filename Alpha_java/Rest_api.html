REST API with Java - A Comprehensive Introduction

What is REST API?

REST (Representational State Transfer) is an architectural style for designing networked applications. It uses HTTP requests to perform CRUD (Create, Read, Update, Delete) operations on resources. REST APIs are lightweight, scalable, and widely used for building web services.

Core Principles of REST

  • Client-Server Architecture: Separation of concerns between client and server
  • Statelessness: Each request contains all information needed; server doesn't store client context
  • Resource-Based URLs: Resources are identified by URIs (e.g., /api/users/123)
  • HTTP Methods: Uses standard HTTP verbs (GET, POST, PUT, DELETE)
  • Representation of Resources: Data typically in JSON or XML format

HTTP Methods in REST

Method Purpose Example
GET Retrieve data from server GET /api/users
POST Create new resource POST /api/users
PUT Update entire resource PUT /api/users/123
DELETE Delete resource DELETE /api/users/123

Building REST APIs with Java

Java provides several frameworks for building REST APIs:

  • Spring Boot: Most popular, with @RestController and @RequestMapping annotations
  • Jersey: Reference implementation of JAX-RS
  • JAX-RS: Java API for RESTful Web Services
  • Quarkus: Modern cloud-native framework

Basic REST API Example with Spring Boot

A simple endpoint that returns user data:

@RestController
@RequestMapping("/api/users")
public class UserController {
    
    @GetMapping("/{id}")
    public User getUser(@PathVariable Long id) {
        return userService.findById(id);
    }
    
    @PostMapping
    public User createUser(@RequestBody User user) {
        return userService.save(user);
    }
    
    @PutMapping("/{id}")
    public User updateUser(@PathVariable Long id, @RequestBody User user) {
        return userService.update(id, user);
    }
    
    @DeleteMapping("/{id}")
    public void deleteUser(@PathVariable Long id) {
        userService.delete(id);
    }
}

Key Benefits of REST APIs

  • Simple and easy to understand
  • Language and platform independent
  • Scalable and stateless design
  • Uses standard HTTP protocols
  • Easy to test and debug
  • Supports caching for better performance

Conclusion

REST APIs have become the standard for modern web development. Java, combined with frameworks like Spring Boot, provides a robust and efficient way to build scalable, maintainable REST services. Understanding REST principles is essential for any backend developer.
